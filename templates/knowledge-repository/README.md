# Team Knowledge

This repository stores reviewed Markdown knowledge as the source of truth.

## Rules

- Use a topic branch and Pull Request for every Entry change.
- Do not commit `.totonoe/` search indexes.
- Run `Totonoe Knowledge: Validate Repository` before review.
- Review credentials, customer data, internal hosts, personal data, classification, and source access.
- Preserve superseded Entries as history instead of deleting or overwriting them.

## Access

Document the project owner, default classification, contributor/reviewer groups, branch protection, backup owner, and offboarding process here before adding confidential data.
