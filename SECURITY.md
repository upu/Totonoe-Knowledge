# Security Policy

## 公開コードと機密データの分離

このリポジトリはTotonoe Knowledgeのソースコードを公開する場所です。利用者が登録するチャット、仕様、顧客情報、接続情報などは公開リポジトリへ保存しないでください。

機密ナレッジには、社内Git、privateリポジトリ、またはアクセス制御されたローカルワークスペースを使用してください。

## 生成モードと外部通信

- `template`: 入力は拡張内で処理され、Language Modelへ送信されない
- `languageModel`: 入力はユーザーが選択したVS Code Language Model Providerへ送信される
- `ask`: 登録ごとに上記を選択する

Language Modelはユーザーが開始した登録操作の中で選択します。Providerの利用条件、保持方針、管理者ポリシーは利用環境ごとに確認してください。

## 秘密情報候補の検査

既定では外部送信前と、ローカルの保存先に関連付けた登録プレビューを開く前に、次のような文字列を検査します。Remoteなど保存先付きプレビューを利用できない環境ではローカル保存直前に検査します。

- 秘密鍵ヘッダー
- GitHubトークン、AWSアクセスキー、JWT
- 認証情報を含むDB URL
- password、token、client secretなどの代入値

警告には種類と件数だけを表示し、検出値そのものをログへ出しません。プレビューを開いた後に利用者が追加した文字列は再検査されない場合があります。検査は補助機能であり、誤検出と見逃しがあります。利用者による確認、Providerポリシー、リポジトリのアクセス制御を置き換えるものではありません。

## その他の原則

- Markdownを正本とし、検索インデックスを再生成可能な派生データとする
- APIキーや認証情報を設定ファイルへ平文保存しない
- テレメトリーや暗黙の外部通信を追加しない
- AI生成内容を自動確定せず、保存前に人が確認できるようにする
- AIに適用バージョンや`supersedes`を推測で確定させず、明示値を保存確認へ表示する
- `repositoryPath`をワークスペース内の相対パスに限定し、外部保存先はフォルダー選択と確認を通じたURIだけを許可する
- 未信頼ワークスペースでは拡張を有効化しない

外部保存先の選択はSettings Syncへ登録しません。選択先へアクセスできなくなった場合は暗黙のフォールバックを行わず、再選択またはワークスペース保存への明示的な切り替えを要求します。詳細は [docs/REPOSITORY_SELECTION.md](docs/REPOSITORY_SELECTION.md) を参照してください。

## ローカル検索インデックス

検索インデックスはワークスペース利用時はワークスペース、外部リポジトリ利用時は選択フォルダーの `.totonoe/index.sqlite` に作成されます。元のMarkdown全文は格納しませんが、ファイルパスと検索可能な1〜3文字の派生トークンを保持するため、元ナレッジと同等の機密性があるものとして扱ってください。

`**/.totonoe/index.sqlite` はこのリポジトリの `.gitignore` に含まれます。利用先リポジトリでも意図せずGit、バックアップ、artifactへ含めないことを確認してください。インデックスは削除してもMarkdownから再生成でき、破損時も正本へ影響しません。

## Remote Repository / MCP

HTTP RepositoryとMCPサーバは未実装です。将来実装するときの必須境界は [Remote Repository / MCP セキュリティモデル](docs/REMOTE_SECURITY_MODEL.md) に定義しています。API契約は [Remote Repository API / MCP契約](docs/REMOTE_REPOSITORY_API.md)、backupとrestoreは [Backup / Export / Restore Runbook](docs/BACKUP_RESTORE.md) を参照してください。これらを満たさない試作サーバへ機密ナレッジを送らないでください。

## 脆弱性の報告

認証情報、顧客データ、非公開ナレッジを公開Issueへ記載しないでください。公開しても安全な最小再現だけをIssueへ記載し、機密情報を伴う場合はリポジトリ所有者へ非公開で連絡してください。
