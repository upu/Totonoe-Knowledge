# Security Policy

## 機密情報の扱い

Totonoe Knowledgeは社外秘・個人情報を含む開発ナレッジでの利用を想定します。次の原則を守ります。

- Markdownの保存先は利用者が管理するローカルまたは社内リポジトリとする
- 外部AI Providerへ送信する前に、Provider名と送信対象を利用者が確認できるようにする
- APIキー、パスワード、秘密鍵、接続文字列などをログへ出力しない
- AIによる秘密情報検出を完全な防止策とみなさない
- 検索インデックスはMarkdownから再生成できる派生データとする

現在のpre-alpha版に外部LLM通信はありません。入力テキストはVS Code拡張内で整形され、設定されたワークスペースへ保存されます。

## 脆弱性の報告

リポジトリがprivateの間は、公開Issueに機密情報を書かず、リポジトリ所有者へ非公開で連絡してください。再現に必要な秘密情報、顧客データ、実運用の認証情報は添付しないでください。
