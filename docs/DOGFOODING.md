# v0.1 Dogfooding Guide

## 目的

10〜30件の実ナレッジを登録し、v0.1が日常的に使えるかを判断します。評価対象は登録の手間、生成内容の修正量、検索での再発見率、メタデータの過不足です。

## データを置く場所

この公開ソースコードリポジトリへ実ナレッジを保存しないでください。次のいずれかを別に用意します。

- 社内Gitの専用privateリポジトリ
- GitHubのprivateリポジトリ
- バックアップ対象になっているローカル専用フォルダー

開発ワークスペースから `Totonoe Knowledge: Select Knowledge Repository Folder` を実行し、そのナレッジ専用リポジトリを選択します。選択せず、ナレッジリポジトリ自体をワークスペースとして開いて `repositoryPath`を設定する従来方式も使用できます。

## 実行手順

1. GitHub Actionsの成功したrunから `totonoe-knowledge-vsix-<commit SHA>` artifactを取得する（またはこのリポジトリで `F5`を使用する）
2. artifactのzipを展開し、VS Codeの `Install from VSIX...` でインストールする
3. 開発ワークスペースを開き、上記のprivate／社内リポジトリを保存先として選択する
4. 調査を終えるたびにCopy Allまたは選択範囲から登録する
5. 別の日に、元の言い回しとは異なる検索語で探す

機密情報を含む可能性がある場合は、まず `template` モードを使います。AI生成を使う場合は、所属組織が許可したLanguage Model Providerとデータ取り扱い条件を確認します。

## 1件ごとの記録

機密内容そのものではなく、次のメトリクスだけをIssue #7へ集計します。

| 項目 | 記録例 |
|---|---|
| 登録方法 | clipboard / selection |
| 生成方法 | template / languageModel |
| 登録所要時間 | 90秒 |
| タイトル修正 | なし / 小 / 大 |
| 本文修正 | なし / 小 / 大 |
| 秘密情報警告 | なし / 正検出 / 誤検出 |
| 後日の検索成功 | 1回目 / 再検索 / 未発見 |
| 不足したメタデータ | version、componentなど |
| 適用バージョン | 未設定 / 設定 / 不明 |

## 完成判定

- [ ] 10〜30件を登録した
- [ ] 登録が日常作業を妨げない
- [ ] 代表的な過去調査を検索で再発見できた
- [ ] AI案を人が無理なく検証できた
- [ ] 公開コードと機密ナレッジを分離できた
- [ ] v0.2へ持ち越す改善点をIssue化した

この判定を満たすまでは、コード上の機能が揃っていてもv0.1マイルストーンは完了としません。
